# Backend Architecture Review Remediation Sprint 01 Admin Auth

Date: 2026-05-03

## Scope

Sprint 1 addresses Finding 7: `/api/v1/admin/queue` was authenticated but did not require an explicit admin principal.

## Changes

- Added role to JWT claims generated by `internal/auth`.
- Added a centralized API principal helper with:
  - user ID;
  - username;
  - role;
  - auth type;
  - optional API key ID.
- Added `adminRequired()` middleware.
- Applied `adminRequired()` to `GET /api/v1/admin/queue`.
- Documented the API-key policy in tests: API keys are authenticated but are not accepted for admin routes.
